# 小市值回测二轮优化设计

**日期**: 2026-03-30

**目标**

在现有 [`scripts/backtest_small_cap_lake.py`](/home/autumn/quant/stock/scripts/backtest_small_cap_lake.py) 基础上，实现一版“混合缓存版”优化策略，覆盖以下四个改进点：

1. Rank Buffer 持仓缓冲区
2. 历史质量过滤（`ROE > 0` + 近似历史 `ST` 过滤）
3. 中证 1000 择时滞后带
4. 个股动量过滤

同时输出消融实验结果，便于比较各项优化的归因。

## 一、 当前基线回顾

现有基线策略已经实现：

- 每周信号、下周开盘调仓
- 行业内 `PE/PB` 低分位过滤
- 小市值排序取前 10
- 次新过滤
- 中证 1000 `MA20` 择时

已验证区间约为 `2010-01-04` 至 `2026-03-13`，但存在：

- 调仓过于刚性，导致高换手
- 无历史 `ROE` / `ST` 风险过滤
- 择时过于敏感，容易来回打脸
- 会捞到仍处于下跌趋势中的“飞刀股”

## 二、 优化目标

### 1. Rank Buffer

将“买入条件”和“卖出条件”解耦。

- 买入：进入候选池后，市值排名前 `10`
- 继续持有：当前持仓只要仍满足：
  - 市值排名前 `30`
  - `pe_rank <= 0.6`
  - `pb_rank <= 0.6`
  - 风险过滤仍通过
  - 动量过滤仍通过
- 卖出：跌出容忍区，或择时切换到风险状态

这个改动不再把每周信号当作“完整持仓快照”，而是作为“允许新买/必须卖出”的边界条件。

### 2. 风险过滤

#### 2.1 ROE

本地 Lake 没有历史 `ROE` 字段，允许使用 Akshare 在线补数，并缓存到本地。

缓存文件：

- `data/cache/small_cap_financial_quality.parquet`

缓存字段：

- `symbol`
- `report_period`
- `announcement_date`
- `roe`
- `source`
- `updated_at`

信号日使用规则：

- 只允许使用 `announcement_date <= signal_date` 的记录
- 每个 `symbol` 取最近一期
- 过滤条件：`roe > 0`

若 Akshare 接口无法直接提供公告日，则采用保守近似：

- 年报：`report_period + 120天`
- 中报：`report_period + 60天`
- 一季报 / 三季报：`report_period + 45天`

不得直接用“报告期对应值”无脑前填，否则会引入未来函数。

#### 2.2 ST 风险过滤

Akshare 当前可用接口中，`stock_zh_a_st_em()` 更接近“当前风险警示板”，不是天然的历史日频 ST 表。

因此这轮采用“近似历史 ST 缓存”：

- 缓存文件：`data/cache/small_cap_risk_flags.parquet`
- 主要来源：
  - `stock_zh_a_st_em()` 当前风险警示板
  - 如可行，再结合名称变更 / 公司公告接口，回溯包含 `ST/*ST/退` 的名称段

缓存字段：

- `symbol`
- `date`
- `is_st_like`
- `reason`
- `source`
- `updated_at`

若无法稳定构造完整历史 ST 曲线，则结果说明中明确标注为“近似 PIT 风险过滤”，不宣称完全无未来函数。

### 3. 择时滞后带

将单阈值 `close >= ma20` 改为双阈值状态机：

- 建仓安全：`close > ma20 * 1.015`
- 清仓风险：`close < ma20 * 0.985`
- 中间区域：保持上一状态

这要求择时函数返回“状态序列”，而不是单日布尔值。

### 4. 个股动量过滤

在信号日横截面上，为每只股票计算：

- 个股 `MA20`
- 条件：`close > ma20_stock`

只有同时满足估值、小市值、风险过滤和动量过滤的股票，才能进入“新买入候选池”。

对已有持仓，继续持有是否要求保留动量过滤，采用保守方案：

- 若个股跌回 `MA20` 下方，则允许卖出，不再强制持有缓冲。

## 三、 数据与缓存设计

### 本地数据

继续使用：

- `v_bar_daily_hfq`
- `v_daily_hfq_w_ind_dim`

### 新缓存

#### 财务质量缓存

文件：`data/cache/small_cap_financial_quality.parquet`

用途：

- 将 Akshare 获取的财务摘要保存为本地可复用缓存
- 避免每轮回测重复请求所有股票

#### 风险标志缓存

文件：`data/cache/small_cap_risk_flags.parquet`

用途：

- 把在线风险警示信息和名称特征整理成信号日可 join 的风险标志

### DuckDB 临时 join 方式

回测脚本运行时：

1. 构造周度候选股票列表
2. 若缓存缺失，则先在线补数并写本地缓存
3. 将缓存 DataFrame 注册为 DuckDB 临时表
4. 在 DuckDB 查询中与本地 Lake 做 `LEFT JOIN`

不要求本轮把两张缓存表正式并入 Lake catalog。

## 四、 策略执行逻辑

### 候选生成

对每个信号日：

1. 从 DuckDB 取该日横截面
2. 计算行业内 `PE/PB` 分位
3. 计算个股 `MA20`
4. join 历史 `ROE` 与风险标志
5. 形成“可买候选池”

### 持仓更新

状态机逻辑：

1. 若大盘风险状态为 `False`，全部清仓
2. 否则：
   - 对现有持仓，逐一判断是否仍在缓冲区内
   - 保留仍合格的股票
   - 在剩余额度中，从“前 10 买入候选池”中补足到目标仓位数

### 仓位

- 默认目标持仓数 `10`
- 等权配置

## 五、 消融实验

同一脚本支持以下模式：

- `baseline`
- `buffer_only`
- `buffer_risk`
- `full_combo`

每个模式单独输出：

- `metrics`
- `trades`
- `weekly_picks`
- `equity`

并汇总成一个总表，比较：

- 总收益
- 年化
- 最大回撤
- 夏普
- 交易次数
- 胜率
- 手续费

## 六、 验收标准

- 能跑通至少四组模式的回测
- 缓存首次创建后，重复运行不重复抓取已存在记录
- Rank Buffer 确实降低交易次数
- 滞后带择时确实变成状态机逻辑，而非简单阈值
- `full_combo` 结果能落盘并可与 `baseline` 对比

## 七、 明确限制

- 历史 ST 过滤本轮为 Akshare 近似缓存，不承诺 100% PIT 精确
- 历史 `ROE` 公告日若接口缺失，则使用保守滞后近似
- 北交所/新上市标的若在线接口财务数据缺失，默认按保守口径剔除
